How It Works

Three-stage pipeline from raw input to fraud decision.

1. Business Inputs

User submits transaction context — amount, device, location, frequency, and risk level through the UI.

2. Feature Engineering

Backend maps business signals into numerical fraud patterns that align with the PCA feature space the model was trained on.

3. ML Prediction

Random Forest runs the engineered features through 100+ decision trees and returns a fraud probability score.

Theory & ML Concepts

What powers this system — explained plainly.

PCA Features (V1–V28)

The Kaggle Credit Card Fraud dataset has 28 anonymized features named V1 to V28. These are the result of Principal Component Analysis — a technique that compresses original transaction data (merchant, time, geography, etc.) into a smaller set of uncorrelated numbers. The bank anonymized it for privacy before releasing it.

Example: V4 = 2.8, V14 = -3.1 might represent an unusually large transaction from a rare merchant category — the model learned this pattern signals fraud.

Class Imbalance Problem

In the real dataset, only 0.17% of transactions are fraud — about 492 out of 284,807. A naive model that predicts "not fraud" every single time would be 99.83% accurate but completely useless. Class imbalance forces us to use precision, recall, and AUC-ROC — not accuracy — to measure how well the model actually catches fraud.

Example: A model catches 90% of fraud cases but also flags 5% of legitimate ones. That tradeoff is a business decision — missing fraud is costlier than a false alarm.

SMOTE — Synthetic Oversampling

SMOTE (Synthetic Minority Oversampling Technique) generates new, synthetic fraud samples by interpolating between real fraud cases in feature space. Instead of just duplicating fraud records, it creates believable variations, giving the model more fraud patterns to learn from without simply repeating the same data.

Example: If two fraud cases have V4 = [2.0, 3.0], SMOTE creates a new sample at V4 = 2.5 — a plausible fraud point the model hasn't seen before.

Random Forest Classifier

Random Forest trains N independent decision trees, each on a random subset of features and rows. At prediction time, every tree votes — fraud or not — and the majority wins. This reduces overfitting that a single tree would have and makes the model robust to noisy, outlier-heavy fraud data.

Example: 100 trees vote → 73 say "fraud", 27 say "legitimate" → fraud probability = 73%. The model doesn't guess; it aggregates evidence.
